## Architecture Overview
The Responsive solution comprises the Java SDK, Control plane, and storage providers as follows which combine as follows:

<div className={'grid lg:grid-cols-2 gap-4'}>
    <div className={'flex flex-col items-center mb-8 aspect-square max-w-[400px] mx-auto'}>
        <img
            alt={"architecture"}
            src={require('/static/img/docs/concepts/architecture/architecture.png').default}
            className={'my-auto'}
        />
    </div>

    <div className={'max-w-4xl mx-auto text-sm flex flex-col justify-center'}>
    | **Component**           | **Description**                                                                                                                                                          |
    |-------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
    | Kafka Provider      | Runs in any environment: self-hosted, cloud (e.g., Confluent, Redpanda, Warpstream), or locally for testing.                                                             |
    | Java SDK            | Extends Kafka Streams with features like remote storage, async processing, and row-level TTL. Fully compatible with the Kafka Streams API; no app code changes required. |
    | Responsive Operator | Manages Kafka Streams apps, scaling them via signals from a Controller in Responsiveâ€™s Control Plane. Deployable via a Helm chart.                                       |
    | Control Plane       | Monitors apps, computes optimal configs for autoscaling, and provides live metrics via a Metrics API. Fully managed by Responsive.                                       |
    | Storage Backend     | Offloads state to distributed databases like MongoDB, ScyllaDB, or RS3. Supports VPC-peered or provider-managed storage. DynamoDB support is coming soon.                |
    </div>
</div>
